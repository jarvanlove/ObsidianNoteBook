## 强制规则：Git 自动提交

### 提交触发时机

**必须提交：**
- 完成一个完整功能块时
- 完成一次问题修复时
- 重要文件变更后

**禁止自动 commit：**
- 除非用户明确说"commit"或调用 `/commit` 技能
- FATAL-005 规定：**Git 操作未经授权，禁止自作主张 commit**

### 提交信息规范

使用约定式提交（Conventional Commits）：

```
<类型>(<范围>): <简短描述>

[可选的正文]

[可选的脚注]
```

### 类型列表

| 类型 | 含义 |
|------|------|
| feat | 新功能 |
| fix | Bug修复 |
| docs | 文档变更 |
| style | 格式调整（不影响功能） |
| refactor | 重构（非新功能非修复） |
| test | 测试相关 |
| chore | 构建/工具变更 |

### 示例

```
feat(科研智能体): 添加文献搜索MCP Server

- 集成 SearXNG 学术搜索
- 支持中文论文库
- 添加搜索结果缓存

Closes #12
```

### 提交检查清单

提交前确认：
- [ ] 测试通过（如果有测试）
- [ ] 没有遗留 `console.log` / `debugger`
- [ ] 提交信息准确描述变更
- [ ] 不包含敏感信息
