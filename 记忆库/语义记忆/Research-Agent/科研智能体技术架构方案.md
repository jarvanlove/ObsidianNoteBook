## 1. 产品定位

### 1.1 产品背景

**愿景**：打造一个**通用科研智能助手**，用户通过自然语言对话框可以完成一切科研任务，AI产出不满足时支持反思修复直到用户满意。

**现有问题**：
- 现有Dify集成的AI功能（对话框、知识库、科技报告、申报书生成）各自独立，体验碎片化
- 每次对话AI没有记忆，做不到真正的智能辅助
- 一次生成不满意，无法让AI反思修复

### 1.2 核心能力

```
┌─────────────────────────────────────────────────────────────┐
│                     通用科研对话框                            │
│                                                               │
│   用户: "帮我综述大语言模型在教育领域的应用"                    │
│                                                               │
│   AI:   "好的，我将帮你完成这个任务..."                        │
│         [自动分解任务 → 调用工具 → 生成综述]                   │
│                                                               │
│   用户: "第三部分不够详细，帮我补充"                           │
│                                                               │
│   AI:   [Critic评审] "确实，我重新生成第三部分..."           │
│         [反思 → 修复 → 直到满意]                              │
│                                                               │
│   用户: "满意了，同步到项目里程碑"                             │
│                                                               │
│   AI:   [自动同步到科研平台指定里程碑]                        │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 技术选型背景

| 现状 | 问题 | 解决方案 |
|------|------|----------|
| Dify单一Agent | 缺乏多Agent协作 | **LangGraph** |
| 无记忆 | 每次对话从零开始 | **Memory + pgvector** |
| 无反思机制 | 产出质量不可控 | **Critic Agent** |
| 各自独立 | 无法自然语言串联 | **通用对话框 + MCP** |

## 3. 新增核心技术详解

### 3.1 LangGraph（多Agent协作框架）

**作用**：替代单一Agent，实现多Agent并行协作 + 状态管理

**为什么必须**：
- Dify的Agent是单一链条，无法并行
- LangGraph支持条件分支、并行执行、循环（反思）
- 每个子Agent专注单一任务，更易维护

**架构示例**：
```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated
import operator

class ResearchState(TypedDict):
    user_request: str
    task_type: str
    context: dict
    result: str
    critique: str
    iteration: int

# 构建图
graph = StateGraph(ResearchState)

# 添加节点
graph.add_node("router", router_agent)
graph.add_node("literature", literature_agent)
graph.add_node("writing", writing_agent)
graph.add_node("analysis", analysis_agent)
graph.add_node("critic", critic_agent)
graph.add_node("memory", memory_agent)

# 定义边
graph.add_edge("router", "literature", condition=lambda s: s["task_type"] == "literature")
graph.add_edge("router", "writing", condition=lambda s: s["task_type"] == "writing")
graph.add_edge("literature", "critic")
graph.add_edge("writing", "critic")
graph.add_edge("critic", END, condition=lambda s: s["result_satisfactory"])

# 如果不满意，循环回对应Agent重新生成
graph.add_edge("critic", "literature", condition=lambda s: not s["result_satisfactory"])
```

### 3.2 Memory System（记忆系统）

**作用**：解决Dify无记忆问题，每次对话都知道"你是谁，你之前做了什么"

**三层记忆**：
```
┌────────────────────────────────────────────┐
│           对话历史 (Redis)                   │
│   - 当前会话上下文                          │
│   - 最近N轮对话                            │
│   - 自动过期 (24小时)                       │
└────────────────────────────────────────────┘
                    │
                    ▼
┌────────────────────────────────────────────┐
│          语义记忆 (pgvector)                │
│   - 用户偏好向量                           │
│   - 历史任务特征                           │
│   - 跨会话长期记忆                         │
└────────────────────────────────────────────┘
                    │
                    ▼
┌────────────────────────────────────────────┐
│         用户Profile (PostgreSQL)            │
│   - 用户基本信息                           │
│   - 常用任务类型                           │
│   - 成果历史                               │
└────────────────────────────────────────────┘
```

**代码示例**：
```python
class MemoryAgent:
    def __init__(self, redis_client, vector_store, pg_pool):
        self.redis = redis_client
        self.vector_store = vector_store
        self.pg = pg_pool

    async def save_context(self, user_id: str, context: dict):
        # 1. 保存到Redis（短期）
        await self.redis.lpush(f"chat:{user_id}", json.dumps(context))
        await self.redis.expire(f"chat:{user_id}", 86400)  # 24小时

        # 2. 生成向量保存到pgvector（长期）
        embedding = await self.embedding_model.embed(context["content"])
        await self.vector_store.insert({
            "user_id": user_id,
            "content": context["content"],
            "embedding": embedding,
            "task_type": context.get("task_type")
        })

    async def get_relevant_memory(self, user_id: str, query: str):
        # 查询相关记忆
        query_embedding = await self.embedding_model.embed(query)
        memories = await self.vector_store.search(
            query_vector=query_embedding,
            filter={"user_id": user_id},
            top_k=5
        )
        return memories
```

### 3.3 Critic Agent（反思评审机制）

**作用**：AI生成后自动评审，不满意则重新生成，直到满意或达到最大轮次

**这是Dify完全没有的能力**：

```
用户输入 ──► 生成Agent ──► Critic Agent评审
                                      │
                           ┌──────────┴──────────┐
                           ▼                     ▼
                      [满意/轮次到]            [不满意+还有轮次]
                           │                     │
                           ▼                     ▼
                       输出给用户          反馈给生成Agent
                                                │
                                                ▼
                                          重新生成
                                                │
                                                └──────► 再次评审
                                                         (最多3轮)
```

**代码示例**：
```python
CRITIC_PROMPT = """你是一个严格的学术评审专家。评审以下AI产出：

任务：{task}
产出：{result}

评审维度：
1. 准确性 - 内容是否准确，有无事实错误？
2. 完整性 - 是否覆盖了任务要求的各个方面？
3. 逻辑性 - 论证是否严密，逻辑是否连贯？
4. 规范性 - 格式是否规范，引用是否正确？

如果产出满足要求，返回：
{"satisfactory": true, "comment": "很好，可以输出"}

如果不满足要求，返回：
{"satisfactory": false, "comment": "问题描述", "suggestion": "修改建议"}
"""

class CriticAgent:
    def __init__(self, llm):
        self.llm = llm

    async def review(self, task: str, result: str) -> dict:
        response = await self.llm.agenerate([
            CRITIC_PROMPT.format(task=task, result=result)
        ])
        return json.loads(response.text)

    async def loop(self, generate_fn, task: str, max_iterations=3):
        """反思循环"""
        for i in range(max_iterations):
            result = await generate_fn()

            review = await self.review(task, result)
            if review["satisfactory"]:
                return result, i + 1  # 返回结果和迭代次数

            # 不满意，带着评审意见重新生成
            result = await generate_fn(feedback=review)

        return result, max_iterations  # 达到最大轮次
```

### 3.4 Redis（会话缓存 + 任务队列）

**作用**：
- 会话上下文缓存（替代Dify的无状态）
- 异步任务队列（Celery后端）

**为什么必须**：
- 文献综述可能需要3-5分钟，不能让HTTP请求一直等待
- 需要WebSocket推送进度给前端

```python
# Celery + Redis 配置
from celery import Celery

celery_app = Celery(
    "research_agent",
    broker="redis://localhost:6379/0",
    backend="redis://localhost:6379/1"
)

@celery_app.task
def literature_review_task(user_id: str, topic: str, session_id: str):
    """异步文献综述任务"""
    # 1. 更新进度到Redis
    redis.set(f"task:{session_id}:progress", "searching")

    # 2. 执行任务...
    papers = search_papers(topic)

    redis.set(f"task:{session_id}:progress", "crawling")
    details = crawl_papers(papers)

    redis.set(f"task:{session_id}:progress", "generating")
    result = generate_summary(details)

    # 3. 保存结果
    redis.set(f"task:{session_id}:result", result)
    redis.set(f"task:{session_id}:status", "completed")

    # 4. WebSocket通知前端
    notify_websocket(session_id, {"status": "completed", "result": result})
```

### 3.5 MCP-Platform（平台功能封装）

**作用**：把Java平台的API封装为MCP工具，让AI可以用自然语言调用平台功能

```
Java平台功能                    MCP-Platform封装
─────────────────────────────────────────────────
项目管理 API        ──►     MCP-Project (创建项目、查询进度)
里程碑 API         ──►     MCP-Milestone (提交成果、查看里程碑)
用户管理 API       ──►     MCP-User (查询用户、权限)
文件管理 API       ──►     MCP-File (上传下载)
仪表盘 API         ──►     MCP-Dashboard (查询统计数据)
```

**代码示例**：
```python
# mcp_server_platform.py - 封装Java平台API
from mcp.server.fastmcp import FastMCP
import httpx

mcp = FastMCP("research-platform")
http_client = httpx.AsyncClient(base_url="http://java-backend:8080")

@mcp.tool()
async def create_milestone(project_id: str, title: str, content: str):
    """在科研平台创建里程碑成果"""
    response = await http_client.post(
        "/api/milestone/create",
        json={
            "projectId": project_id,
            "title": title,
            "content": content,
            "status": "PENDING_REVIEW"
        }
    )
    return response.json()

@mcp.tool()
async def query_project_progress(project_id: str):
    """查询项目进度"""
    response = await http_client.get(f"/api/project/{project_id}/progress")
    return response.json()

@mcp.tool()
async def get_user_info(user_id: str):
    """获取用户信息"""
    response = await http_client.get(f"/api/user/{user_id}")
    return response.json()
```

### 3.6 Re-Ranker（检索结果重排）

**作用**：向量检索后的结果相关性重排，提高准确率

**为什么需要**：
- 单纯向量检索可能返回看似相关但实际不准确的结果
- Re-Ranker模型可以进一步排序

```python
from sentence_transformers import CrossEncoder

reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")

async def search_with_rerank(query: str, papers: list[dict]):
    # 1. 向量检索
    vector_results = await vector_store.search(query, top_k=20)

    # 2. Re-Rank重排
    pairs = [(query, paper["content"]) for paper in vector_results]
    scores = reranker.predict(pairs)

    # 3. 按分数排序
    ranked = sorted(zip(vector_results, scores), key=lambda x: x[1], reverse=True)

    return [item[0] for item in ranked[:10]]
```

### 7.2 接口设计

```python
# Python Agent服务 API
from fastapi import FastAPI, WebSocket, BackgroundTasks
from sse_starlette.sse import EventSourceResponse

app = FastAPI()

@app.post("/api/agent/chat")
async def chat(request: ChatRequest):
    """
    通用对话接口
    - 支持流式SSE输出
    - 自动路由到对应Agent
    - 支持反思循环
    """
    session_id = request.session_id or generate_uuid()

    async def event_generator():
        async for chunk in agent.chat_stream(
            user_id=request.user_id,
            message=request.message,
            session_id=session_id
        ):
            yield {
                "event": "message",
                "data": json.dumps(chunk)
            }

    return EventSourceResponse(event_generator())

@app.post("/api/agent/milestone/sync")
async def sync_to_milestone(request: MilestoneSyncRequest):
    """
    同步到里程碑
    """
    result = await platform_agent.create_milestone(
        project_id=request.project_id,
        title=request.title,
        content=request.content
    )
    return result

@app.websocket("/ws/agent/{session_id}")
async def websocket_endpoint(websocket: WebSocket, session_id: str):
    """
    WebSocket实时通信
    - 任务进度推送
    - 生成完成通知
    """
    await websocket.accept()
    subscriber = redis.pubsub(f"session:{session_id}")
    await subscriber.subscribe()
    async for message in subscriber.listen():
        await websocket.send_json(json.loads(message["data"]))
```

## 9. 技术选型理由

### 9.1 为什么选 LangGraph？

- **多Agent原生支持**：条件边、并行边、循环边（反思）直接支持
- **状态管理**：自动传递State，无需手动管理上下文
- **调试友好**：可视化执行链路
- **与LangChain生态兼容**：继续用LangChain的工具和Prompt

### 9.2 为什么必须加Memory？

- Dify每次对话从零开始，用户需要重复解释背景
- Memory让AI"认识"用户，知道用户做什么研究
- 跨会话记忆可以关联历史任务，形成完整科研轨迹

### 9.3 为什么必须加Critic？

- 一次生成不可控，用户可能需要多次手动修改
- Critic让AI自己检查问题，带着意见重新生成
- 减少用户参与度，提升最终产出质量
