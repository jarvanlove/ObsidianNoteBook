## 使用说明

以后你要让 AI 基于这套知识库设计 agent，不要直接丢一句模糊需求。  
先按这个模板补齐关键信息，再和知识库文档一起喂给 AI。

这样做的好处是：

- AI 更容易准确拆解需求
- AI 更容易映射成 Tool / Task / Agent / Skill / Plugin
- AI 更容易给出可落地的 runtime 设计

## 模板正文

```text
# 业务需求输入

## 1. 背景
- 我当前要解决的问题是什么：
- 这个需求属于什么场景：
  - coding
  - research
  - sales
  - support
  - ops
  - content
  - knowledge
  - 其他：

## 2. 目标
- 最终用户想达到什么结果：
- 成功标准是什么：

## 3. 输入
- 用户会提供什么输入：
- 系统可用的已有数据源有哪些：
- 是否需要访问外部系统：

## 4. 输出
- 我希望系统最终输出什么：
- 输出给谁：
- 输出是一次性结果还是持续任务：

## 5. 流程特征
- 这是单轮任务还是长流程：
- 是否需要后台运行：
- 是否需要多个 agent 协作：
- 是否需要验证环节：
- 是否需要人工审批：

## 6. 风险与约束
- 是否涉及：
  - 文件修改
  - shell 执行
  - 生产环境操作
  - 对外发送消息
  - 数据写入
  - 敏感信息
- 是否存在组织或合规约束：

## 7. 恢复与审计
- 是否要求可 resume：
- 是否要求任务进度可见：
- 是否要求保留审计记录：

## 8. 前端壳
- 我希望从哪里触发：
  - terminal
  - web
  - IDE
  - API
  - 其他：
- 我希望如何看到状态和结果：

## 9. MVP 边界
- 第一阶段必须做什么：
- 第一阶段明确不做什么：

## 10. 其他补充
- 特别偏好的设计方式：
- 现有技术栈：
- 必须兼容的系统：
```
