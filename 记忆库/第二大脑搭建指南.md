## 1. 系统概述

### 什么是第二大脑系统？

这是一个让 Claude Code 拥有长期记忆、主动行为能力的系统。无论你在哪个项目工作，Claude 都能：

- 记住你之前的决策和偏好
- 自动追踪任务和时间意图
- 系统化吸收外部知识
- 主动提醒和归档

### 核心特性

- **跨项目生效**: 在任何目录打开 Claude Code 都能使用
- **三层记忆**: 情景记忆 → 语义记忆 → 强制规则
- **共享存储**: 所有记忆放在一起，Obsidian 按需筛选
- **数据主权**: 所有数据本地存储，纯 Markdown
- **Obsidian 可视化**: 用 Obsidian 查看和编辑知识图谱

## 4. Windows 搭建步骤

### 步骤 1: 创建目录结构

```powershell
mkdir -p "C:\Work\note\ObsidianNoteBook\记忆库\情景记忆\2026-03"
mkdir -p "C:\Work\note\ObsidianNoteBook\记忆库\语义记忆\科研智能体"
mkdir -p "C:\Work\note\ObsidianNoteBook\记忆库\语义记忆\通用"
mkdir -p "C:\Work\note\ObsidianNoteBook\记忆库\语义记忆\第二大脑"
mkdir -p "C:\Work\note\ObsidianNoteBook\记忆库\强制规则"
```

### 步骤 2: 创建基础文件

**记忆库 README** — `C:\Work\note\ObsidianNoteBook\记忆库\README.md`:
```markdown
# 第二大脑 记忆库 v2.0

> 统一存储，多项目自动路由，按时间归档

## 架构

- 情景记忆/ — 对话历史（按月归档）
- 语义记忆/ — 知识提炼
- 强制规则/ — AI 行为约束

## 固定路径

- Windows: `C:\Work\note\ObsidianNoteBook\记忆库\`
- macOS: `~/Documents/ObsidianNoteBook/记忆库/`
- 任务清单: `C:\Work\note\ObsidianNoteBook\记忆库\任务清单.md`
```

**任务清单** — `C:\Work\note\ObsidianNoteBook\记忆库\任务清单.md`:
```markdown
# 任务清单

## 🔴 紧急重要

## 🟡 重要不紧急

## 🟢 日常事项

## ✅ 已完成
```

### 步骤 3: 确认全局 CLAUDE.md

检查 `C:\Users\你的用户名\.claude\CLAUDE.md` 是否包含第二大脑系统指针章节（v2.0）。
如果没有，从 `记忆库/第二大脑搭建指南.md` 中复制相应的章节到 CLAUDE.md 末尾。

任务清单路径已更新为：`C:\Work\note\ObsidianNoteBook\记忆库\任务清单.md`
3. 询问从哪个任务开始
```

### 步骤 4: 初始化 Git

```powershell
cd "C:\Work\note\ObsidianNoteBook"
git init
echo ".DS_Store`n*.tmp`n*.temp`n.obsidian/workspace*" > .gitignore
git add 记忆库/
git commit -m "init: 第二大脑系统 v2.0"
```

### 步骤 5: 在 Obsidian 中打开

1. 打开 Obsidian
2. 点击"打开文件夹作为 Vault"
3. 选择 `C:\Work\note\ObsidianNoteBook`

## 6. 使用方法

### 常用指令

| 你说的话 | Claude 做什么 |
|---------|-------------|
| **"启动第二大脑"** | 读取任务清单，显示今日概览 |
| **"明天要做XX"** | 自动添加到 🟡 重要不紧急 |
| **"必须做XX"** | 自动添加到 🔴 紧急重要 |
| **"记录这个决策"** | 创建情景记忆（自动打项目标签） |
| **"总结成经验"** | 更新语义记忆（自动打项目标签） |
| **"看看这篇文章+链接"** | 启动6步知识吸收 |
| **"完成了XX"** | 移到 ✅ 已完成 |

### 情景记忆 vs 语义记忆 分类指南

#### 核心区别

| 维度 | 情景记忆 | 语义记忆 |
|------|---------|---------|
| **目的** | 记录"那次发生了什么" | 提炼"以后遇到类似怎么做" |
| **形式** | 事件 + 时间 + 上下文 | 原则 + 模式 + Checklist |
| **复用性** | 低（特定上下文） | 高（抽象提炼） |

#### 情景记忆适用场景

| 内容类型 | 示例 |
|---------|------|
| 决策记录 | 选了 A 方案没选 B 的原因 |
| 需求变更 | 客户要求砍掉 X 功能的讨论 |
| Bug 根因 | 为什么这个 bug 存在 |
| 里程碑达成 | v1.0 准时上线 |
| 踩坑记录 | 踩了哪些坑及解决过程 |
| 特定偏好 | 老板/客户的特定要求 |
| 团队冲突 | 冲突及解决过程 |

#### 语义记忆适用场景

| 内容类型 | 示例 |
|---------|------|
| 技术选型原则 | 为什么选 PostgreSQL 不用 MySQL |
| 代码规范 | Git commit 规范、命名规则 |
| 架构模式 | 微服务划分原则 |
| 团队协作流程 | Code review 流程 |
| API 设计原则 | RESTful 规范 |
| 测试策略 | 何时写单元测试/集成测试 |

#### 快速判断法

```
能做成模板/原则/Checklist？ → 语义记忆
                          不能 → 情景记忆

以后会遇到类似场景？       → 语义记忆
                          不会 → 情景记忆
```

#### 注意：项目产物不属于记忆库

开发文档、PRD、开发计划等是**项目产物**，应放在项目目录本身。

### 项目记录格式（重点）

在不同项目中使用第二大脑时，记忆以以下格式记录：

**情景记忆文件命名**：
```
记忆库/情景记忆/YYYY-MM/MMDD-项目名-事件.md
```
示例：
```
记忆库/情景记忆/2026-03/0321-科研智能体-技术架构讨论.md
记忆库/情景记忆/2026-03/0321-电商项目-数据库选型.md
```

**情景记忆文件格式**：
```markdown
# #项目标签（自动添加）

## 事件标题

内容正文...

- 日期: YYYY-MM-DD
- 项目: #项目名
```

**语义记忆文件命名**：
```
记忆库/语义记忆/<中文项目名>/主题.md
记忆库/语义记忆/通用/主题.md           ← 通用知识
```
示例：
```
记忆库/语义记忆/科研智能体/LangGraph技术选型.md    ← 文件内标签：#科研智能体
记忆库/语义记忆/科研智能体/PostgreSQL最佳实践.md
记忆库/语义记忆/通用/Python最佳实践.md            ← 通用知识，无项目标签
```

**注意**：语义记忆用中文目录名组织，文件内标签用中文项目名（如 `#科研智能体`），而非英文目录名。

**完整使用示例**：
```
你在 科研智能体 项目中工作：
1. 对 Claude 说"记录我们决定用 pgvector 的决策"
2. Claude 自动：
   - 工作目录识别 → #科研智能体
   - 创建文件 → 记忆库/情景记忆/2026-03/0321-科研智能体-pgvector决策.md
   - 文件头部写 → # #科研智能体
   - 写入内容...
3. 之后在 Obsidian 中搜索 #科研智能体 → 找到所有该项目记忆
```

### 与 Obsidian 配合

**Obsidian 的作用**:
- 可视化浏览知识图谱
- 使用双向链接 `[[笔记名]]` 建立关联
- 用文件夹视图按月查看情景记忆
- 用 Dataview 查询按标签筛选
- 用 `#项目名` 标签区分项目

**Claude Code 的作用**:
- 自动化记录和分类
- 任务追踪和提醒
- 知识吸收和提炼
- 跨项目一致性记忆

### 工作流程

```
在任意项目工作
    ↓
告诉 Claude 记录决策/任务
    ↓
Claude 写入记忆库/任务清单
    ↓
打开 Obsidian 查看和整理
    ↓
建立笔记之间的链接
    ↓
下次工作时 Claude 读取记忆
```

## 7. 加载规则与写入规则

### 加载规则（v2.0 自动路由）

每次启动 Claude Code 时，自动执行以下流程：

```
1. 识别当前项目（从工作目录路径提取项目标识）
   ↓
2. 加载强制规则（6条，始终）
   ↓
3. 加载任务清单
   ↓
4. 优先加载项目专属记忆（有 #当前项目 标签）
   ↓
5. 再加载通用记忆（无标签）
```

**项目识别示例**：
| 工作目录 | 识别结果 |
|---------|---------|
| `...\SALL-ZOE\Link-AI-Sci-Z-Platform-College` | `#科研智能体` |
| `...\SALL-ZOE\Personal-Assistant` | `#SALL-ZOE-个人助手` |
| `...\FreelanceWork` | `#FreelanceWork` |
| `...\ObsidianNoteBook` | `#第二大脑` |
| 其他 | `#通用` |

**加载反馈**（简短）：
```
[第二大脑 v2.0] 项目: #科研智能体 | 任务: 2进行中 | 记忆: 已加载
```

### 写入规则（v2.0 自动打标签）

#### 任务写入流程

```
用户说"要做XX"
    ↓
判断优先级
    ├── "必须/紧急" → 🔴 紧急重要
    ├── "明天/以后" → 🟡 重要不紧急
    └── "有空时" → 🟢 日常事项
    ↓
写入任务清单.md 对应区域
    ↓
告知用户存储位置
```

#### 记忆写入流程（自动打标签）

```
用户说"记录XX"
    ↓
识别当前项目（从工作目录）
    ↓
判断类型
    ├── 对话/事件/决策 → 情景记忆
    ├── 方法/技术/知识 → 语义记忆
    └── 行为约束 → 强制规则
    ↓
自动在文件头部添加 #当前项目 标签
    ↓
按格式写入对应目录
```

**写入格式示例**：
```markdown
# #科研智能体

pgvector 优于自建向量服务...
```

**特殊指令**：
- 用户说"这是通用知识" → 写入时不加项目标签（所有项目可见）

#### 任务写入流程

```
用户说"要做XX"
    ↓
判断优先级
    ├── "必须/紧急" → 🔴 紧急重要
    ├── "明天/以后" → 🟡 重要不紧急
    └── "有空时" → 🟢 日常事项
    ↓
写入任务清单.md 对应区域
    ↓
告知用户存储位置
```

#### 记忆写入流程

```
用户说"记录XX"
    ↓
判断类型
    ├── 对话/事件/决策 → 情景记忆
    ├── 方法/技术/知识 → 语义记忆
    └── 行为约束 → 强制规则
    ↓
按格式写入对应目录
    ↓
告知用户存储位置
```

#### 外部知识吸收流程

当用户提供链接时，执行6步：
1. **理解** — 提取核心观点
2. **路由** — 决定存储位置
3. **提炼** — 转化为可复用格式
4. **存储** — 写入对应记忆目录
5. **关联** — 检查相关记忆并建立链接
6. **反馈** — 告知用户摘要和位置

## 8. 常见问题

### Q1: 为什么不做项目物理隔离？

**A**: 记忆的本质是时间序列，按项目物理隔离会：
- 同一技术在不同项目重复记录
- 跨项目知识无法自然复用
- 记忆库越来越碎片化

用 Obsidian 的文件夹/标签/Dataview 可以随时按项目筛选，统一存储更灵活。

### Q2: 历史记录和记忆库有什么区别？

**A**:
- **历史记录**: 被动记录所有对话
- **记忆库**: 主动分类和提炼的知识

建议先用记忆库，如果需要自动记录，再添加历史收集脚本。

### Q3: 强制规则可以修改吗？

**A**: 可以。当现有规则不适用时：
1. 在对应规则文件中更新内容
2. 在文件顶部添加 `updated: YYYY-MM-DD`
3. 告知用户变更内容

### Q4: 如何备份？

**A**: Git 定期提交：
```bash
git add 记忆库/
git commit -m "backup: YYYY-MM-DD"
git push
```

### Q5: Windows 路径有中文可以吗？

**A**: 可以，但建议使用英文路径避免编码问题。如果必须用中文，确保文件保存为 UTF-8。

### Q6: 多台电脑如何同步？

**A**: 将 `ObsidianNoteBook` 目录放入云同步盘（iCloud、OneDrive、Dropbox），或使用 Git 推送到远程仓库。
