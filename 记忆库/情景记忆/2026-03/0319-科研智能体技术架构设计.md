## 技术选型决策（已确认）

### 向量数据库
**选择: pgvector**
- 理由: 已有PostgreSQL，复用现有资源

### MCP Server
**选择: Phase1用fastmcp，Phase2可切换官方SDK**
- fastmcp: 上手快，适合演示
- 官方SDK: 生产级，稳定

### Agent框架
**选择: LangGraph DeepAgent + LangGraph**
- 成熟度高，国产LLM适配好
- 支持多Agent协作

### 工具链
| 工具 | 用途 |

### LLM
**选择: Kimi / Qwen / MiniMax**
- 国产模型，客户接受度高

## Dify现有资产处理

### Phase 1.5 关键决策

| 现有功能 | 处理策略 |
|---------|---------|
| 知识库 | MCP封装Dify API，不迁移数据 |
| 申报书生成 | 迁移到LangGraph新Agent |
| 红头文件解析 | 迁移到新Agent + MCP封装 |
| 科技报告生成 | 迁移到LangGraph新Agent |

### 核心原则
- Dify投入的工作流不会浪费
- 通过MCP封装调用现有Dify能力
- 新Agent逐步替代/增强现有功能
