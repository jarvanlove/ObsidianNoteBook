
## 三层记忆

| 层级 | 内容 | 写入时机 |
|------|------|---------|
| 情景记忆 | 对话、决策、事件 | 重要对话后、关键决策后 |
| 语义记忆 | 方法论、技术总结、知识提炼 | 从经验中提炼出可复用模式时 |
| 强制规则 | AI 行为约束（FATAL 铁律） | 系统级规则设定时 |

---

## 情景记忆 vs 语义记忆 分类指南

### 核心区别

| 维度 | 情景记忆 | 语义记忆 |
|------|---------|---------|
| **目的** | 记录"那次发生了什么" | 提炼"以后遇到类似怎么做" |
| **形式** | 事件 + 时间 + 上下文 | 原则 + 模式 + Checklist |
| **复用性** | 低（特定上下文） | 高（抽象提炼） |
| **典型内容** | 决策过程、踩坑记录、会议结论 | 技术选型原则、团队规范 |

---

### 情景记忆适用场景（事件驱动、上下文强）

| 内容类型 | 示例 |
|---------|------|
| 决策记录 | 选了 A 方案没选 B 的原因 |
| 需求变更 | 客户要求砍掉 X 功能的讨论 |
| Bug 根因 | 为什么这个 bug 存在 |
| 里程碑达成 | v1.0 准时上线 |
| 踩坑记录 | 踩了哪些坑及解决过程 |
| 特定偏好 | 老板/客户的特定要求 |
| 团队冲突 | 冲突及解决过程 |
| 项目历程 | 重要阶段事件 |

> **判断技巧**：需要还原"当时发生了什么"吗？→ 情景记忆

---

### 语义记忆适用场景（原则驱动、可复用）

| 内容类型 | 示例 |
|---------|------|
| 技术选型原则 | 为什么选 PostgreSQL 不用 MySQL |
| 代码规范 | Git commit 规范、命名规则 |
| 架构模式 | 微服务划分原则 |
| 团队协作流程 | Code review 流程 |
| API 设计原则 | RESTful 规范 |
| 需求分析框架 | 原型优先开发流程 |
| 测试策略 | 何时写单元测试/集成测试 |
| 沟通模式 | 周会节奏、决策层级 |

> **判断技巧**：能做成模板/原则/Checklist 吗？→ 语义记忆

---

### 快速判断法

```
这个能做成模板/原则/Checklist？
  能 → 语义记忆
  不能 → 情景记忆

这个以后会遇到类似场景吗？
  会 → 语义记忆
  不会 → 情景记忆

这个需要还原"当时发生了什么"吗？
  需要 → 情景记忆
  需要指导未来行动 → 语义记忆
```

---

### 注意：项目产物不属于记忆库

开发文档、PRD、开发计划等是**项目产物**，应放在项目目录本身，而非记忆库：

```
项目目录/
├── docs/              ← PRD、开发文档
├── plans/             ← 开发计划
└── 记忆库/             ← 第二大脑记忆
```

**记忆库存储的是"从项目经历中提炼的洞见"，而非项目原始文档。**

## 加载规则（每次会话自动执行）

```
1. 识别当前项目（从工作目录路径）
   ↓
2. 加载强制规则（6条，始终）
   ↓
3. 加载任务清单
   ↓
4. 优先加载项目专属记忆（#当前项目 标签）
   ↓
5. 再加载通用记忆（无标签）
```

## 使用方法

### 启动

在任何项目工作目录对 Claude Code 说：

```
启动第二大脑
```

→ 自动识别项目 → 加载强制规则 → 加载任务清单 → 加载该项目记忆

---

### 记录类

| 你说的话 | Claude 做什么 |
|---------|-------------|
| `记录这个决策` | 创建情景记忆（自动打 `#当前项目` 标签） |
| `总结成经验` | 创建语义记忆到该项目专属目录（自动打标签） |
| `这是通用知识` | 创建语义记忆到 `通用/`（无项目标签） |
| `看看这篇文章+链接` | 启动 6 步知识吸收流程 |

**情景记忆命名**：`记忆库/情景记忆/YYYY-MM/MMDD-项目-事件.md`
**语义记忆命名**：`记忆库/语义记忆/<项目名>/主题.md`

---

### 任务类

| 你说的话 | Claude 做什么 |
|---------|-------------|
| `必须做XX` | 加入 🔴 紧急重要 |
| `明天要做XX` | 加入 🟡 重要不紧急 |
| `有空时做XX` | 加入 🟢 日常事项 |
| `完成了XX` | 移到 ✅ 已完成 |

---

### 查询类

| 你说的话 | Claude 做什么 |
|---------|-------------|
| `启动第二大脑` | 读任务清单，显示今日概览 |
| `查看#某项目的记忆` | 筛选该项目所有记忆 |
| `看看有什么任务` | 显示任务清单 |

---

### GitHub 提交规则

- **触发词**：`commit` / `提交` / `/commit`（被动，等你指令）
- **禁止自动提交**（FATAL-005）
- **只提交**：`记忆库/`（含所有记忆、任务清单、搭建指南、源文）

---

## 自动行为

- 检测到时间意图 → 自动添加任务到对应优先级
- 每周日 → 提醒归档已完成任务
- 任务过期 → 提醒并询问是否调整优先级
